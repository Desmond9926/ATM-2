建议协议格式：JSON 文本 + TCP
每条消息一行或加长度头都可以。课程项目里最简单的是“每次发送完整 JSON 字符串”。

统一结构建议如下。
1. 请求报文通用格式
{
  "action": "login",
  "sessionId": "",
  "data": {
  }
}
字段说明：
+ action
操作类型
+ sessionId
登录成功后由服务端返回，后续请求携带；未登录时可为空
+ data
具体业务字段

2. 响应报文通用格式
{
  "success": true,
  "code": 0,
  "message": "操作成功",
  "data": {
  }
}
字段说明：
+ success
是否成功
+ code
业务状态码
+ message
提示信息
+ data
返回数据

3. 协议接口明细
3.1 注册/开户
请求：
{
  "action": "register",
  "sessionId": "",
  "data": {
    "cardNo": "62220001",
    "userName": "ZhangSan",
    "password": "123456",
    "accountNo": "ACC10001",
    "initialBalance": 1000.00
  }
}
响应：
{
  "success": true,
  "code": 0,
  "message": "开户成功",
  "data": {
    "userId": 1
  }
}

校验规则：
1. cardNo 唯一
2. accountNo 唯一
3. 初始金额不能小于 0
4. 密码长度建议至少 6 位
---
3.2 登录
请求：
{
  "action": "login",
  "sessionId": "",
  "data": {
    "cardNo": "62220001",
    "password": "123456"
  }
}
成功响应：
{
  "success": true,
  "code": 0,
  "message": "登录成功",
  "data": {
    "sessionId": "S_100001",
    "userId": 1,
    "userName": "ZhangSan",
    "role": "user"
  }
}
失败响应：
{
  "success": false,
  "code": 1002,
  "message": "密码错误，还剩2次机会",
  "data": {
    "remainingAttempts": 2
  }
}
锁定响应：
{
  "success": false,
  "code": 1003,
  "message": "账户已锁定",
  "data": {
    "isLocked": true
  }
}
业务逻辑：
1. 先查 cardNo
2. 再查是否锁定
3. 比对哈希密码
4. 失败累计次数
5. 达到 3 次锁定
6. 成功则清零并生成 sessionId

4. 建议状态码
0      成功
1001   用户不存在
1002   密码错误
1003   账户已锁定
1004   未登录或会话失效
1005   权限不足
1006   参数错误
1007   余额不足
1008   目标账户不存在
1009   不能转账给自己
1010   账户状态异常
1011   数据库错误
1012   旧密码错误
1013   卡号已存在
1014   账户号已存在
