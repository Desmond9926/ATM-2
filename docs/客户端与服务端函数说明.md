# 客户端与服务端函数说明

## 说明

本文档整理了 ATM 项目中客户端与服务端主要函数的基本信息，重点包括：

- 函数所在头文件或源文件
- 函数参数
- 实现功能
- 返回结果

说明范围以 `client/` 和 `server/` 为主，便于阅读项目结构和理解各模块职责。

## 一、客户端函数说明

## 1. `ClientMain.cpp`

### `void ConfigureConsoleEncoding()`

- 所在文件：`client/ClientMain.cpp`
- 参数：无
- 功能：将 Windows 控制台输入输出编码切换为 `UTF-8`，减少中文乱码问题。
- 返回结果：无

### `int main()`

- 所在文件：`client/ClientMain.cpp`
- 参数：无
- 功能：
  - 配置控制台编码
  - 创建 `ClientSocket` 对象
  - 连接服务端 `127.0.0.1:8888`
  - 创建 `ClientApp` 并进入客户端主循环
- 返回结果：
  - `0`：正常退出
  - `1`：连接服务端失败

## 2. `ClientSocket`

头文件：`client/ClientSocket.h`

### `ClientSocket()`

- 所在头文件：`client/ClientSocket.h`
- 参数：无
- 功能：构造客户端套接字对象，初始化内部 `socket_`。
- 返回结果：构造函数，无显式返回值

### `~ClientSocket()`

- 所在头文件：`client/ClientSocket.h`
- 参数：无
- 功能：析构时关闭连接并释放 Winsock 资源。
- 返回结果：析构函数，无显式返回值

### `bool Connect(const std::string& ip, int port)`

- 所在头文件：`client/ClientSocket.h`
- 参数：
  - `ip`：服务端 IP 地址
  - `port`：服务端端口号
- 功能：初始化 Winsock 并连接到服务端。
- 返回结果：
  - `true`：连接成功
  - `false`：连接失败

### `void Close()`

- 所在头文件：`client/ClientSocket.h`
- 参数：无
- 功能：关闭当前 socket 连接并清理资源。
- 返回结果：无

### `bool Send(const std::string& text)`

- 所在头文件：`client/ClientSocket.h`
- 参数：
  - `text`：要发送给服务端的文本内容
- 功能：通过 TCP 将请求文本发送到服务端。
- 返回结果：
  - `true`：发送成功
  - `false`：发送失败

### `bool Receive(std::string& text)`

- 所在头文件：`client/ClientSocket.h`
- 参数：
  - `text`：输出参数，用于接收服务端返回文本
- 功能：从 TCP 连接中读取服务端响应。
- 返回结果：
  - `true`：接收成功
  - `false`：接收失败

### `bool SendRequest(const std::string& requestText, std::string& responseText)`

- 所在头文件：`client/ClientSocket.h`
- 参数：
  - `requestText`：请求报文
  - `responseText`：输出参数，用于保存响应报文
- 功能：组合执行发送和接收，完成一次完整请求。
- 返回结果：
  - `true`：请求成功并收到响应
  - `false`：请求失败

## 3. `RequestHandler`

头文件：`server/RequestHandler.h`

### `RequestHandler(AuthService* authService, AccountService* accountService, TransactionService* transactionService)`

- 所在头文件：`server/RequestHandler.h`
- 参数：
  - `authService`：认证服务对象
  - `accountService`：账户服务对象
  - `transactionService`：交易服务对象
- 功能：创建请求分发器并保存服务依赖。
- 返回结果：构造函数，无显式返回值

### `std::string HandleRequest(const std::string& requestText, const std::string& clientIp)`

- 所在头文件：`server/RequestHandler.h`
- 参数：
  - `requestText`：客户端原始请求文本
  - `clientIp`：客户端 IP
- 功能：解析请求文本，调用 `Dispatch` 分发到具体业务函数，再序列化为响应字符串。
- 返回结果：响应报文字符串

### `Response Dispatch(const Request& request, const std::string& clientIp)`

- 所在头文件：`server/RequestHandler.h`
- 参数：
  - `request`：已解析请求对象
  - `clientIp`：客户端 IP
- 功能：根据 `action` 字段调用对应业务函数，如注册、登录、查询余额、转账、管理员功能等。
- 返回结果：统一响应对象 `Response`

## 4. `AuthService`

头文件：`server/AuthService.h`

### `explicit AuthService(Database* db)`

- 所在头文件：`server/AuthService.h`
- 参数：
  - `db`：数据库对象指针
- 功能：创建认证服务对象。
- 返回结果：构造函数，无显式返回值

### `Response RegisterUser(const Request& request)`

- 所在头文件：`server/AuthService.h`
- 参数：
  - `request`：注册请求，包含卡号、用户名、密码、账户号、初始金额

## 三、补充说明

### 1. 统一请求与响应对象

客户端和服务端很多函数都依赖 `Request` 与 `Response`：

- `Request`：表示客户端请求，包含 `action`、`sessionId`、`data`
- `Response`：表示服务端响应，包含 `success`、`code`、`message`、`data`

相关定义位于：`common/Protocol.h`

### 2. 重要业务关系

- `ClientApp` 负责采集输入并发起请求
- `ClientSocket` 负责网络通信
- `TcpServer` 负责接收连接和读写 TCP 数据
- `RequestHandler` 负责按 `action` 分发业务
- `AuthService` 负责认证、会话、管理员功能
- `AccountService` 负责余额、存款、取款
- `TransactionService` 负责转账和流水查询
- `Database` 负责 MySQL 访问
- `PasswordUtil` 负责盐值与密码哈希

### 3. 当前安全控制

以下操作已要求用户在已登录基础上再次输入密码：

- 查询余额
- 存款
- 取款
- 转账

这样即使会话已经登录，也仍需进行二次密码验证后才能执行资金类或敏感操作。
