服务端

  • TcpServer：监听端口、处理客户端连接
  • RequestHandler：解析请求并按 action 分发
  • AuthService / AccountService / TransactionService：业务核心

9. 关键代码：请求分发

server/requesthandler.cpp routing protocol 
std::string RequestHandler::HandleRequest(const std::string& requestText, const std::string& clientIp) {
    Request request;
    if (!ProtocolUtil::ParseRequest(requestText, request)) {
        Response response;
        response.success = false;
        response.code = 1006;
        response.message = "请求格式错误";
        return ProtocolUtil::BuildResponse(response);
    }

    return ProtocolUtil::BuildResponse(Dispatch(request, clientIp));
}

Response RequestHandler::Dispatch(const Request& request, const std::string& clientIp) {
    if (request.action == "login") return authService_->Login(request, clientIp);
    if (request.action == "query_balance") return accountService_->QueryBalance(request);
    if (request.action == "deposit") return accountService_->Deposit(request);
    if (request.action == "withdraw") return accountService_->Withdraw(request);
    if (request.action == "transfer") return transactionService_->Transfer(request);

    Response response;
    response.success = false;
    response.code = 1006;
    response.message = "未知操作";
    return response;
}

这一层把 TCP 文本请求转成统一的 Request 对象，并按 action 分发到认证、账户和交易服务，保证协议入口统一。

10. 关键代码：二次密码验证

server/accountservice.cpp security 2fa like check 
bool AccountService::VerifySessionPassword(const SessionInfo& session,
                                           const Request& request,
                                           Response& errorResponse) {
    const auto passwordIt = request.data.find("password");
    if (passwordIt == request.data.end() || passwordIt->second.empty()) {
        errorResponse = MakeResponse(false, 1006, "缺少密码");
        return false;
    }

    std::vector<std::string> row;
    const std::string sql =
        "SELECT password_hash, password_salt FROM users WHERE id = " +
        std::to_string(session.userId) + " LIMIT 1";

    if (!db_->Query(sql) || !db_->FetchRow(row)) {
        errorResponse = MakeResponse(false, 1001, "用户不存在");
        return false;
    }

    if (!PasswordUtil::VerifyPassword(passwordIt->second, row[1], row[0])) {
        errorResponse = MakeResponse(false, 1012, "密码验证失败");
        return false;
    }

    return true;
}

余额查询、存款、取款、转账都不是只依赖会话，而是要求用户再次输入密码，降低“已登录状态被误用”的风险。

11. 关键代码：转账事务

server/transactionservice.cpp transaction consistency 
if (!db_->BeginTransaction()) {
    return MakeResponse(false, 1011, db_->GetLastError());
}

if (!db_->Execute("UPDATE accounts SET balance = " + std::to_string(sourceAfter) + " WHERE id = " + std::to_string(source.id)) ||
    !db_->Execute("UPDATE accounts SET balance = " + std::to_string(targetAfter) + " WHERE id = " + std::to_string(target.id)) ||
    !InsertTransaction(source.id, "transfer_out", amount, source.balance, sourceAfter, target.accountNo, remark) ||
    !InsertTransaction(target.id, "transfer_in", amount, target.balance, targetAfter, source.accountNo, remark)) {
    db_->Rollback();
    return MakeResponse(false, 1011, db_->GetLastError());
}

if (!db_->Commit()) {
    db_->Rollback();
    return MakeResponse(false, 1011, db_->GetLastError());
}

转账的关键不是“扣款”或“入账”单独成功，而是两边余额更新和两条流水写入必须处于同一个事务里。

12. 关键代码：密码哈希

server/passwordutil.cpp bcrypt sha-256 
std::string PasswordUtil::HashPassword(const std::string& password, const std::string& salt) {
    return Sha256(password + salt);
}

bool PasswordUtil::VerifyPassword(const std::string& inputPassword,
                                  const std::string& salt,
                                  const std::string& storedHash) {
    return HashPassword(inputPassword, salt) == storedHash;
}

项目没有明文存储密码，而是把随机盐和哈希结果分开保存。登录和二次验证时都通过重新计算哈希进行比对。

13. 关键业务规则

  • 登录前先检查账户是否锁定
  • 密码错误一次，failed_attempts + 1
  • 连续错误 3 次，自动设置 is_locked = 1
  • 登录成功后清零失败次数
  • 所有资金类操作都要写入 transactions
  • 转账必须使用数据库事务保证扣款和入账一致
  • 查询余额、存款、取款、转账前需要再次输入密码

14. 安全设计

密码安全

  • 注册时随机生成盐值
  • 保存 SHA256(password + salt)
  • 数据库不保存明文密码

敏感操作防护

  • 登录后使用内存会话控制访问
  • 资金类操作要求二次密码验证
  • 登录日志和管理员日志可追踪

15. 转账一致性设计

校验会话与密码 → 检查目标账户 → 检查余额 → 开启事务 → 扣减源账户 → 增加目标账户 → 写两条流水 → 提交事务

任何一步失败都执行回滚，保证“要么全部成功，要么全部不生效”。
